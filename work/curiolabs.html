The Brief

Most digital tools are built for markets that aren't Africa. The pricing models, the assumptions, the interfaces — all calibrated for a different context. CurioLabs was founded on a simpler idea: build the tools that actually need to exist here, and build them properly.

Each tool started from a real frustration — a process that was broken, information that was scattered, a job that required too much manual effort. The brief was never abstract. It was always: what is the specific thing that needs fixing?

The Work

The platform comprises several tools, each with its own scope and audience — but a consistent design language across all of them. Clean interfaces, fast inputs, clear outputs. No unnecessary complexity.

VehicleIQ — a vehicle specification lookup tool built for DVLA registration staff and the automotive industry. Surfaces OEM colour codes, engine variants, and trim details in seconds.

GhanaDuty — an import duty calculator for vehicles entering Ghana. Breaks down every tax component line by line, so buyers understand what they'll pay before they commit.

Quoted — a pricing intelligence tool for African designers. Calibrated to local markets, local rates, and the specific dynamics of freelancing on the continent.

Dispatch — a file export automation planner for designers and studios. Generates structured handoff plans so deliveries are consistent and professional every time.

Each tool was designed, built, and deployed independently. The stack is intentionally lightweight: HTML, CSS, JavaScript, no build tooling where it isn't necessary. Performance is a design decision.

The Outcome

All four tools are live and in active use. VehicleIQ solves a daily problem inside DVLA operations. GhanaDuty is used by buyers, clearing agents, and dealers who want accurate numbers before calling anyone. Quoted is helping designers across Ghana, Nigeria, and Kenya understand what their work is worth.

CurioLabs is proof that one designer with a clear process can build tools that function at a professional standard — without a team, without a budget, and without waiting for someone else to solve the problem first.
